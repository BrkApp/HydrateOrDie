# HydrateOrDie - Assistant Guide

## Commands
- Run: `flutter run`
- Test: `flutter test` (Unit/Widget) or `flutter test integration_test`
- Analyze: `flutter analyze`
- Format: `dart format .`
- Build Runner: `dart run build_runner build --delete-conflicting-outputs`

## Architecture & Rules (Strict)
- **Pattern:** Clean Architecture (Presentation -> Domain -> Data).
- **State Management:** Riverpod.
- **DI:** GetIt.
- **Persistence:** Sqflite (Local), Firebase (Remote).
- **Testing:** Domain >= 80%, Data >= 70%, Presentation >= 50%.
- **Files:** Snake_case filenames. PascalCase classes.
- **Docs:** Dartdoc mandatory for public members.
- **Deps:** NO new packages without explicit PM validation.

## Git & Workflow
- **Commits:** `[EPIC-X.Y] Description` (Atomic commits).
- **Branches:** `feature/epic-X-story-Y-desc`.
- **Validation:** Check `docs/definition-of-done.md` before finishing.
- **Reports:** Save `completion-report.md` & `dod-report.md` in `docs/stories/epic-X/reports/` (NOT root).

## Structure
- `lib/core`: Constants, themes, utils, DI.
- `lib/data`: Models (DTOs), Repositories Impl, Data Sources.
- `lib/domain`: Entities, Repositories Interfaces, Use Cases.
- `lib/presentation`: Screens, Widgets, Providers.

## Critical
- NEVER modify `docs/governance.md` or `docs/architecture.md` without explicit permission.
- ALWAYS run tests before finishing a task.
- Handle all errors (Try/Catch) - No raw crashes.
